# Dependency Policy

## Goals

MediaKit aims to keep dependencies:
- Minimal
- Stable
- Well-maintained
- Android-focused

## Rules

- Avoid heavy unnecessary libraries
- Prefer AndroidX and Kotlin-first solutions
- Minimize transitive dependencies
- Keep modules independently usable

## Priorities

Higher priority than adding dependencies:
- Stability
- Performance
- Memory safety
- Build speed
- SDK size

## Long-Term Direction

MediaKit should remain lightweight and modular even as the ecosystem grows.
