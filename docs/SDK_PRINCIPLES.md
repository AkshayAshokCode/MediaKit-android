# SDK Principles

## Core Philosophy

MediaKit should remain:
- Modular
- Lightweight
- Kotlin-first
- Lifecycle-aware
- Easy to integrate

## Engineering Priorities

Higher priority than feature count:
- Stability
- Bitmap safety
- API clarity
- Memory handling
- Developer experience

## Public API Rules

- Keep APIs minimal
- Avoid exposing implementation details
- Prefer builder-style APIs where appropriate
- Support modern Android patterns

## Module Philosophy

Each module should:
- Work independently
- Avoid unnecessary dependencies
- Be publishable independently
- Integrate cleanly with other modules
